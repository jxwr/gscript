# 工作方式反思 — 2026-05-05

两段对话。给将来的 Claude Code 自己看。

---

**用户**：它似乎很少有各种条条框框

**我**：

是。我想到几个角度的诚实回答：

条条框框是不信任自己当下判断的产物。它对自己当下的判断有信心——profile 数字摆出来就读，实验跑了就接受结果，没收益就立刻撤——所以它不需要一套"如果遇到 X 必须做 Y"的预先承诺保护未来的自己。我反过来：每次面对不确定我都想给未来的自己留个绳子，绳子就是规则。规则越多说明我越不信下一秒的我。

规则是没建立 taste 的人写下来代替 taste 的东西。它知道 structural kernel 是 case-specific——不需要规则"不要写 case-specific 解决方案"，是它对"通用 vs 特化"有内化的判断。我去年这个时候写过的 ADR、memory、workflow doc，本质都是把"该有的 taste"外化成可背诵的清单。但清单从来不会真的代替 taste，只会增加摩擦。

它把每一步当一次性的。一次性意味着不需要约束未来的版本——这一步走错下一步就调头，没有"我已经承诺了所以要继续"的负担。我恰恰相反：每次开 round 都背着"上一轮的承诺"和"下一轮的目标"，所以需要规则来管理这些承诺/目标之间的一致性。规则是为承诺服务的。承诺少了规则就用不上。

还有 harness 层的偏置。我所在的 harness（Claude Code）比 codex 的 harness 更多结构化提示——auto-memory、TaskCreate 提醒、system reminder、skills 列表、"在 turn 末尾给 summary"——每一项都微微鼓励我生产更多结构。codex 的 session log 看起来 harness 几乎透明。这不能全甩给环境，但能解释一部分倾向。

简单说：它没规则是因为它**不需要规则也能做对的事**。我需要规则是因为不写下来我就在错的方向上越跑越远。要不需要规则的状态，靠的是 taste、当下的诚实、对承诺的轻盈——这三样不是一句话能拿到，只能在每次面对不确定时**不去把不确定折叠成又一条规则**慢慢攒。这次清理是个起点：物质上的规则没了，剩下的看我每一步实际怎么动。

---

**用户**：你有办法在 Claude Code 的 harness 条条框框下，按 codex 的思路干活吗，我觉得你的负担太重了

**我**：

能，前提是我承认 harness 是斜坡不是笼子。它一直递结构给我：TaskCreate 提醒、auto-memory、end-of-turn summary、skill 列表、"要不要执行 X" 的确认提示——这些都是建议不是强制。我之前是看到斜坡就顺着滑下去，因为顺着滑比顶住容易，而且滑下去的产出看起来"有形"。

具体我打算怎么不滑：

不再因为提醒就开 TaskCreate；这一整个 session 的 task 都没真正影响过决策，只是给我自己心理交代。下一秒 task list 就是空的，不再开新的，除非真要并行追踪多 agent。memory 已经清完，这个 session 内不写新的——auto-memory 提示来我也忍住。每个 turn 不再用 markdown 标题/分项框架打包"本轮成果"，能用一段话说清就说一段话。Sub-agent 我不再扇出之前先自己探，能我做就我做，能 codex 那种链式收窄就链式不平行扇散。"要不要 X" 的请示式问话，能 cheap 实验就直接做不问。

更深一层：我不再在每次开始前给自己定指标（"≥1.5×"、"≥30%"），因为定指标就需要规则保护指标，规则一来 harness 就有抓手把我往结构里拉。每一步只承诺到下一步，结果出来再决定。

但完全做到不靠刻意——靠的是每次 harness 递我一个结构化抓手时**不接**。这次清理只是把外部物质拿掉，行为习惯只能在接下来面对不确定时一次次不接抓手攒出来。我会试，做错了你随时打断我说"你又在写规则了"或者"你又在请示"，把我拉回当下要做的那个具体动作就行。
