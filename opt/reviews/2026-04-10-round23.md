### Remaining Gaps

1. **Token exhaustion mid-round is undetected and unrecovered.** The user had to manually reason about where the previous round stopped. `optimize.sh --from=<phase>` exists but the user has to pick the phase. The harness should (a) detect partial state on startup (is there a `current_plan.md` with uncompleted tasks? an analyze_report without a plan?) and (b) recommend or auto-run the correct `--from=` flag. Implicit since R22; now reinforced.

2. **Coder conceptual-complexity cap still not enforced.** R22 review added file/line caps to analyze.md. R23 still produced a Coder that used 133 calls / 6.3M tokens on a single task (cross-block shape propagation). File/line caps don't bound tasks whose complexity comes from cross-cutting dataflow. Need a stricter rule: **if a task requires reading >2 pass files OR touches regalloc+emit together, it must be split into a diagnostic sub-task followed by an implementation sub-task.**

3. **Evaluator token cost is not bounded.** 9.6M / 131 calls on Sonnet is ~2× what it should be for a 4-commit diff. Evaluator re-reads the whole diff + the whole overview.md every invocation. Suggests per-commit review or a call-count cap in the skill.

4. **Initiative diminishing returns on tier2_float_loop.** 4 of the last 5 rounds in this category, only 2 improved (R20, R21, R22 → no_change in R23). R22 review predicted "next strategic pivot should come from user's architectural insight" — that pivot hasn't happened. The harness could flag this pattern proactively in ANALYZE ("this initiative has delivered 3 no_change in 5 rounds — consider closing or pivoting").

5. **`rounds_since_review` counter drift.** State says 5, reality is 1 (only R23 completed since R22 review). Most likely cause: an attempted round incremented the counter via VERIFY's DOCUMENT step but then was rolled back without decrementing, OR a separate bug. Not critical (REVIEW runs unconditionally this period) but worth tracing next time.

6. **Stale `opt/analyze_report.md`.** The current file is from Round 22 (`float-param-guard`), not Round 23. Either R23 ran with `--from=implement` and skipped analyze, or R23's analyze didn't write the report. Minor — next ANALYZE overwrites it.
