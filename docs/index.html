The Call Could Not Be Replayed

Tier 2 direct BLR entries are now published only when caller-boundary replay is safe. A native caller cannot resume a callee at the callee's own exit point; replaying the whole call after a visible side effect duplicates program state. The new replay-safety analysis keeps unsafe side-effect-then-exit functions callable through the normal TieringManager path, while preserving raw recursive wins such as ackermann.

2026-04-28
The Coroutine Stayed On The Stack

coroutine_bench was dominated by runtime handoff cost, not Tier 2. Long-lived coroutines now keep their child VM frame paused at coroutine.yield, write resume values back into the saved yield result slots, and continue run() without a goroutine/channel ping-pong. After hardening cached yield, nested resume, Value.Ptr(), and child VM cleanup semantics, the focused guard moved from about 1.6s to 0.03s.

2026-04-28
The Registers Were There

After scalar promotion, nbody.advance still had five hot float spill slots. Tier 2 now allocates from D4-D11,D16-D23 and keeps promoted float phis resident, while field access code reuses existing TypeTable proofs instead of repeating table checks. The production dump moved from five spills to zero, and the focused guard moved nbody from about 0.077s to 0.064s on the merge machine.

2026-04-28
The Exit Edge Finally Split

nbody scalar promotion was blocked by a production CFG shape, not by the idea itself. The pass can now split a shared loop-exit edge, place promoted field writebacks on the isolated path, and preserve field-cache metadata on synthetic materialization operations. The production test now asserts the benchmark pipeline keeps float fields in SSA instead of merely observing a synthetic pass result.

2026-04-28
The Upvalue Was Not An Exit

closure_bench was not waiting for Tier 2. Its hot accumulator closure was stuck behind an over-broad Tier 1 direct-entry replay-safety gate. Valid GETUPVAL and SETUPVAL slots cannot exit, so they no longer block native BLR direct entry after a side effect; callees that perform a real exit-capable operation after the upvalue mutation remain gated. The accumulator section dropped from about 0.050s to 0.008s, and the full benchmark returned to the 0.028s baseline.

2026-04-28
The Load That Did Not Replay The Write

Typed table-array loads can now miss without replaying earlier writes. The cold path reconstructs the original GetTable from data -> header -> table, executes it through TableOpGetTable, and resumes after the lowered load. Remaining representation mismatches use precise interpreter resume at the source PC, while the exit-resume checker treats the load result as an output rather than a pre-exit live input. This is replay infrastructure, not a headline speedup, but it lets future table optimizations avoid a coarse mutation veto.

2026-04-28
The Recurrence Split Before It Overflowed

fibonacci_iterative was not fixed by hard-coding Fibonacci. Tier 2 now recognizes a narrow shift-add recurrence shape, runs the raw-int prefix entirely in registers, and switches to a float continuation exactly when the int48 representation stops being valid. The callee publishes a boxed leaf direct entry instead of pretending to satisfy the raw-int peer ABI, and the benchmark driver can tier up only when its native callees make that profitable. The focused guard moved the benchmark to about 0.025s while preserving the overflow boundary cases.

2026-04-28
The Allocation Exit Became a Cache

Matmul's table exits were not one problem. Typed dense NewTable sites now refill a bounded per-site cache, mixed dense stores such as c[i] = row infer array preallocation before feedback arrives, and a second post-LICM load-elimination pass removes duplicated table-array facts after hoisting. Exits fell from the old hundreds into the low dozens, but wall time stayed near 0.09s, making the next bottleneck the native success path rather than fallback traffic.

2026-04-27
The Load Split Into Four

Dense table loads are no longer one opaque GetTable. Tier 2 lowers monomorphic array reads into header, length, data, and element-load nodes so load elimination and LICM can share or hoist the stable pieces. The same round caught a serious gcd wrong-code: direct Tier 2 entry assumed B0 was the semantic entry after LICM had moved the real entry to B3. The fix routes native entries through Function.Entry and hardens raw-int kernel forcing before the next peer-call ABI step.

2026-04-27
The Globals Learned Their Indexes

Tier 2 globals now use a VM-owned indexed array guarded by structural version. Direct BLR and tail calls publish each callee's const-index map through FuncProto, so recursive callees no longer fall back to global exits just because the caller had a different constant pool. Top-level global reductions get a native write path with map synchronization, while callee writes stay on exit-resume until the broader sync protocol exists.

2026-04-27
The Mutation Stayed Behind the Gate

Quicksort remains blocked, but the recursive table-mutation gate now classifies recovery shapes instead of treating every SetTable alike. Idempotent overwrites are replay-safe; swaps are only read-backed and stay diagnostic-only until the compiler can prove all guards before mutation or record undo metadata. The same round also moved concat and simple string formatting onto lower-overhead runtime paths.

2026-04-27
The Table Was Born Typed

Fresh dense tables can now become typed arrays before feedback catches up. Tier 1 append stores extend typed arrays when capacity is available, and Tier 2 infers local int/float/bool table hints from typed stores. Focused diagnostics now show sieve around 0.029s, matmul around 0.121s, and table_array_access around 0.035s while keeping the direct-deopt protocol conservative.

2026-04-27
The Row Was a Table

Matmul's outer mixed-array loads return row tables, but Tier 2 was not carrying that result type through the nested access path. Tier 1 now records FBTable for mixed-array GETTABLE results, GraphBuilder marks those loads as TypeTable, and mixed-array codegen emits the table guard inline. This is a prerequisite for row-header hoisting: the compiler can now prove that a[i] and b[k] are row tables.

2026-04-27
The Two Pointers That Met in the Middle

Fannkuch's regression came from boxing the two-pointer reversal loop even though lo < hi proves the single-step updates safe: lo + 1 cannot exceed the raw int48 maximum and hi - 1 cannot go below the minimum. Range analysis now recognizes only that strict converging shape, with a negative test for hi -= 2. Seven-run guard moved fannkuch from 0.062s to 0.052s while sort and table-array access stayed flat.

2026-04-27
The Frame We Did Not Write

Raw peer calls were already passing integer arguments in registers, but the success path still pre-boxed those raw args into the callee VM window for fallback. That made fallback simple and the common path slower. The new convention keeps raw peer success register-only and materializes the boxed VM call frame only if fallback or exit-resume needs it. A machine-code test now scans the raw peer shim and verifies there are no mRegRegs stores before BLR.

2026-04-27
The Table Remembered Its Largest Key

Tier 1 already knew table value kinds; it did not remember how large integer-key arrays became. The new per-PC TableKeyFeedback records the largest non-negative integer key observed at table access sites, then Tier 2 folds that into typed preallocation hints with a capped, lazy mixed-array capacity path. Sieve exits dropped into the low 40s while table-array and matmul stayed inside the guard thresholds.

2026-04-27
The Induction That Stayed Raw

Sieve's inner loop was still boxing loop-carried integer arithmetic because overflow safety treated too many recurrences as unsafe. The new rule keeps only guarded forward linear inductions raw: a loop-header phi bounded by < or <=, updated by a loop-invariant non-negative AddInt. Multiplicative and decrementing recurrences stay boxed. Sieve moved from about 0.055s to 0.050s without reopening the overflow-deopt bug class.

2026-04-27
The Debugger That Runs Production

The next optimization loop needed production-grade visibility before more performance work. Six tools now answer the questions that used to require guesswork: -jit-timeline for tier chronology, -jit-dump-warm for real warm Tier 2 artifacts, structured optimization remarks, IR/ASM/source mapping, -exit-stats for hot deopt and exit sites, and GSCRIPT_EXIT_RESUME_CHECK=1 for executable exit-resume invariants. The goal is not prettier dumps. The goal is that every performance premise has to survive production evidence before code changes begin.

2026-04-26
The Call Boundary

Ackermann went from "JIT slower than VM" to a clear Tier 2 win once the raw-int self-recursive ABI became a real codegen contract instead of an analysis note. The fix was not a trace JIT and not an Ackermann special case: prove the static self-recursive integer shape, pass raw args in X0..X3, return raw in X0, materialize boxed fallback frames only when needed, and keep the boxed VM ABI separate from the raw self-call ABI. The remaining LuaJIT gap is now mostly recursive frame and call-boundary overhead.

A Fifty-Percent Regression Nobody Saw

R34's sanity check surfaced a +50.79% cumulative drift on object_creation that no single round had flagged. R35 ran git bisect across seven code-changing commits and converged in four steps on 39b5ef3 — the Shape system rewrite that added a write-only shape *Shape GC pointer to every Table struct and uncapped ScanGCRoots to scan the full register slice. 2026-04-12
The One-Line Gate

R33 was supposed to be a one-line fix to R32's float-gate bug in LoopScalarPromotionPass. The plan cited two files and two lines, both correct: pass_scalar_promote.go:99 gates on GetField.Type == TypeFloat, graph_builder.go:669 emits every production GetField with TypeAny and a trailing GuardType float. Walk the consumer list, classify via the GuardType, done. The Coder wrote the production-pipeline diagnostic test first (which we've been promising since R31), confirmed the failing pre-fix baseline of 9 unpromoted pairs / 0 float phis, applied the plan's exact fix, re-ran the test, and got back bit-identical 9 pairs / 0 phis. The float gate was never the first filter. Two upstream gates bail before classification even happens: an exit-block-preds check that rejects the j-loop because its exit target is the i-loop header (which unavoidably has the i-loop preheader as a co-pred), and isInvariantObj which correctly rejects the second i-loop's b := bodies[i] because the object is genuinely loop-variant. The plan proved the float gate is broken — true — but not that it's the only broken gate — false. Five rounds running I've done this exact thing: find one root cause, cite the line, ship confident, watch it no-op. No production commit. Test kept as observe-only template. Handed to VERIFY as data-premise-error. Next round pivots away from nbody.

The Field That Stayed in a Register

R32 went after nbody's j-loop, the three bi.vx/vy/vz fields that get read, modified, and written every iteration. Classic mem2reg territory — promote the loop-carried fields into an SSA phi at the loop header, turn six field ops per iteration into zero, cut about 78 ARM64 instructions and 36 LDR/STR out of the inner loop. Wrote LoopScalarPromotionPass, 264 lines, wired after LICM, all tests green, evaluator PASS, one Coder attempt. Ran the benchmarks. nbody: 0.248s → 0.248s. Zero. My first instinct was to reach for R23's lesson — M4 superscalar hides memory savings, the dependency chain was already overlapping the arithmetic, write it up as no_change and move on. I almost did. Then I re-ran the diagnostic on the post-pipeline IR. All nine loop-carried pair candidates were still sitting in B2 and B6, untouched. The pass hadn't run. One line, pass_scalar_promote.go:99: if instr.Type == TypeFloat. Production IR emits GetField : any with a trailing GuardType float, never a typed GetField. The unit tests hand-built TypeFloat directly, so they passed through a gate the production pipeline can never hit. R31 was the same shape one round ago. R33 is a one-line gate fix; R32's actual deliverable is the harness rule: every new Tier 2 pass needs a post-pipeline diagnostic re-run test, not just synthetic IR.

The Phi That Ate Sieve

R31 started with a user directive to stop grinding fib and attack sieve. A diagnostic dump of sieve's Tier 2 IR showed three self-referential phis for loop-invariant values — table, step, and n — sitting in the inner-loop header and forcing per-iteration spill-slot round trips on the back edge. Classic redundant-phi SCC, the exact case Braun Algorithm 5 is designed to clean up. Implemented SimplifyPhisPass: 226 lines of Tarjan SCC over the phi subgraph, cardinality-1 collapse, path-compressed resolve across previously-rewritten chains. Full test suite green, evaluator pass, every box ticked. Sieve: 0.085s → 0.084s. Below the 5% floor. Why? The diagnostic was produced by the simplified profileTier2Func pipeline, which is known-stale and has been drifting out of sync with compileTier2 for 25 rounds. Production Tier 2 already collapses those phis via tryRemoveTrivialPhi plus ConstProp/DCE; the SCC I went hunting for does not exist in the real IR. The pass runs and finds nothing to do. R19 may have been the same bug. Two rounds burned on the same bad tool — fixing it is the first line of R32.

2026-04-07
Four Guards, Fourteen Times

nbody’s j-loop has 431 ARM64 instructions and 29 float ops. Four type guards on loop-invariant values run five million times each but can never fail. We hoisted them to the pre-header and propagated shape verification across block boundaries. Instruction count went down. Wall time didn’t move. The M4’s branch predictor was already making the guards free. The lesson: on a wide superscalar core, removing bandwidth-bound overhead does nothing when latency is the bottleneck.
